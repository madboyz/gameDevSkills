# 修订数据包

## 云端简报模板

从本地上下文上升到云端推理时使用：

```markdown
目标：

文件范围：

已验证事实：

约束：

期望输出：

问题：
1.
2.
3.
```

## 审批门控

仅当全部门控通过时才允许编辑：

| 门控 | 检查项 |
| --- | --- |
| 可追溯性 | 每条建议都能映射到源文件或数据包事实 |
| 安全性 | 不向云端发送机密、凭证或无关载荷 |
| 明确性 | 编辑需指明工作表、章节、行或段落 |
| 可验证性 | 本地检查可在应用后确认结果 |

## 评审台账

分开保存已接受与已拒绝的提案：

```json
[
  {
    "id": "chg-001",
    "status": "accepted",
    "target": "Sheet1!reward_value",
    "reason": "",
    "source": "",
    "risk": "low"
  }
]
```

## 提示规则

- 显式询问假设
- 涉及权衡时询问备选方案
- 要求简洁推理，而非链式思维叙述
- 编辑表格时要求机器可读差异

## 本地合并规则

小批量应用云端提案。每批完成后，先比对实际文件结果与请求差异，再继续下一批。
